# Feature Spec — Project Intelligence Framework

## Purpose

Project Intelligence Framework transforms raw project data into structured investment intelligence.

Traditional property platforms only store project listings.

This system converts projects into **investment assets with structured intelligence**.

---

# Core Concept

Instead of storing only basic project information, each project must contain structured intelligence layers.

Project Data
↓
Project Intelligence
↓
Investment Insight
↓
Matching Engine

---

# Intelligence Layers

The framework consists of the following layers:

1. Project Identity
2. Location Intelligence
3. Market Intelligence
4. Risk Intelligence
5. Investment Thesis
6. Investment Grade

---

# 1. Project Identity

Basic project information.

Fields may include:

- project_name
- developer
- city
- district
- product_type
- price_range

This layer defines the base identity of the asset.

---

# 2. Location Intelligence

Measures the quality and potential of the project location.

Attributes:

- distance_to_cbd
- metro_access
- highway_access
- infrastructure_pipeline
- neighborhood_quality

This layer helps the system evaluate long-term growth potential.

---

# 3. Market Intelligence

Captures the market dynamics around the project.

Examples:

- average_price_area
- price_growth_3y
- rental_yield
- supply_level
- demand_level

This layer provides context for investment evaluation.

---

# 4. Risk Intelligence

Evaluates potential risks associated with the project.

Risk categories:

- legal_risk
- construction_risk
- supply_risk
- market_risk

Each risk category may have a score.

---

# 5. Investment Thesis

Human-readable strategic explanation of why the project is attractive.

Examples:

- strong infrastructure development
- high rental demand
- undervalued entry price

This section provides advisory context.

---

# 6. Investment Grade

Projects are assigned a grade based on scoring models.

Example grading:

A — strong investment potential  
B — moderate investment potential  
C — speculative opportunity  
D — high risk

The grading engine already exists in the database layer.

---

# Integration with Matching Engine

Matching Engine uses the intelligence layers to compute:

Project Fit Score

This ensures recommendations are based on structured intelligence rather than raw listings.

---

# UX Integration

Project Intelligence powers the following interfaces:

Opportunity Board  
Project Detail Page  
Project Comparison  
Decision Workspace

Users should see insights instead of raw data fields.

---

# Design Principle

Project Intelligence must always support the advisory-first philosophy.

Projects should be presented as **investment opportunities**, not as property listings.
